BFC的那些事儿

 从接触前端的很长一段时间内，笔者对css的认识都停留在一个非常浅的层面，就是这个一眼看去很简单实则水很深的东东，总是在很细微的地方给我造成各种困扰。一些bug，一些解决方案，看上去莫名其妙，其实当了解了css的原理以后也就豁然开朗了。最近正好在看css2的W3C标准，就谈谈BFC这个神奇的东西吧。

W3C标准中对BFC的定义如下所示：

块格式化上下文 Block formatting contexts
 有这几种框会为其内容创建新的块格式化上下文：浮动框、绝对定位框、非块框的块容器、overflow属性非visible的。
 在块格式化上下文中，框会从包含块的顶部开始，一个接一个地,垂直向下地摆放。两个兄弟框之间的垂直距离由margin属性来决定。在同一个块格式化上下文中，相邻的块级框之间的垂直外边距会出现折叠。
 在块格式化上下文中，每个框的左外边距边要紧贴其包含块的左边。即使在有浮动的情景下也是如此，除非框创建了一个新的块格式化上下文(在这种情况下该框可能会为了避开浮动框而变窄)。

 不知道有没有同学和笔者一样，初看这段话觉得云里雾里的。犹记得当年，笔者在各个博客中见到对BFC的解释都引用了这段话，读了好多遍都没看懂，一度以为是自己认知能力有问题。直到某天我看了W3C标准的前后文，才突然顿悟，之前的打开方式一直有问题嘛~个人认为要理解BFC，首先需要了解可替换元素，框（Box），包含块（containing block）以及格式化上下文（Formatting context）的概念。

可替换元素

 一个 内容 不受CSS视觉格式化模型控制，CSS渲染模型并不考虑对此内容的渲染，且元素本身一般拥有固有尺寸（宽度，高度，宽高比）的元素，被称之为可替换元素。除置换元素外的所有元素都是非置换元素。

框（Box）

 Box 是 CSS 布局的对象和基本单位， 直观点来说，就是一个页面是由很多个 Box 组成的。元素的 display 属性，决定了这个 Box 的类型。 不同类型的 Box， 会参与不同的 Formatting Context（一个决定如何渲染文档的容器），因此Box内的元素会以不同的方式渲染。框大致可以分为以下几种：

块级框:display属性为block, list-item, table 的元素，也就是块元素会生成block-level box。并且参与块级格式上下文(BFC)；除了table框，和可替换元素，一个块级框同时也是一个块容器框，一个块容器框要么只包含块级框，要么创建一个行格式化上下文而只包含行内级框。并非所有的块容器框都是块级框：不可替换的行内块和不可替换的table cell也是块容器但不是块级框。是块级框的块容器称作块框。所以BFC定义中非块框的块容器就是指不可替换的行内块和table cell了。这也就是为什么可以设置display为inline-block或者table-cell来触发一个新的BFC的原因。

行级框:display属性为inline, inline-block, inline-table的元素，会生成inline-level box。并且参与 行级上下文（IFC）；

插入框: run-in box，css3 中的定义，此处不赘述。

包含块

 在CSS 2.1中，许多框的位置和大小都根据一个名为包含块的矩形框边缘来相对计算得到。为元素生成的框通常会充当其后裔框的包含块；我们称框为其后裔“创建”（establishes）了包含块。“框的包含块”表示“框所处的包含块”，而不是其产生的包含块。一个元素的包含块的定义如下:

  1. 根元素存在的包含块称为初始包含块。
  • 在(X)HTML中，根元素是html元素（尽管有的浏览器会不正确地使用body元素）。
  • 而初始包含块的direction属性与根元素相同。(direction属性指定了块的基本书写方向，它还规定了表格列布局的方向、水平溢出的方向等。)
  • 初始包含块的宽度可以由根元素的width属性指定。如果该属性取值为“auto”，用户端提供初始宽度（如，视口的当前宽度）。
  • 初始包含块的高度可以由根元素的height属性指定。如果该属性取值为“auto”，包含块的高度将调整以适应文档内容（即由其内容决定其高度）。
  • 初始包含块不可以被定位或浮动（即，用户端忽略根元素的position和float属性）。
  1. 对于其它元素：如果该元素的定位（position）为“relative（相对）”或者“static（静态）”，它的包含块由它最近的块级、单元格（table cell）或者行内块（inline-block）祖先元素的内容框创建。
  2. 如果元素设定了“固定定位（position: fixed）”，包含块由视口创建。
  3. 如果元素设定了“绝对定位（position: absolute）”，包含块由最近的position属性为“absolute”、“relative”或者“fixed”的祖先元素创建，方法如下：
  • 如果祖先元素是行内元素，包含块取决于祖先元素的direction属性：
    1. 如果direction为“ltr”，包含块的顶、左边是该祖先元素创建的第一个框的顶、左补白边，它的底、右边是该祖先元素创建的最后一个框的底、右补白边。
    2. 如果direction为“rtl”，包含块的顶、右边是该祖先元素创建的第一个框的顶、右补白边，它的底、左边是该祖先元素创建的最后一个框的底、左补白边。
  • 否则，祖先的补白边形成包含块。
  • 如果不存在这样的祖先元素，则元素的包含块为初始包含块。

BFC的规则及应用

 说了这么多，让我们再来整理一下W3C标准中关于BFC的描述，可以抽出定义，布局规则以及何种情况触发BFC这三部分：

定义:

 BFC是一个独立的渲染区域，并且其中元素的布局不受外界的影响。

BFC的布局规则：

  1. 框会从包含块的顶部开始，一个接一个地,垂直向下地摆放。
  2. Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠。
  3. 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  4. 计算BFC的高度时，浮动元素也参与计算

哪些情况会生成BFC

  1. 根元素；
  2. 浮动元素，即float属性非none；
  3. 绝对定位框，position属性为fixed或者absolute；
  4. 非块框的块容器，display为inline-block, table-cell, table-caption, flex, inline-flex；
  5. over-flow不为visible；
